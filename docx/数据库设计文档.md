| 版本 | 日期 | 描述 |
|------|------|------|

## 1. 概要结构设计

### 1.1 数据库概述

| 属性 | 值 |
|------|-----|
| 数据库名称 | medical_cloud_film |
| 数据库类型 | SQL Server 2019 |
| 字符集 | UTF-8 |
| 主键策略 | 系统模块：AUTO INCREMENT；业务模块：雪花ID |

### 1.2 模块划分

数据库表按功能分为两大模块：

| 模块前缀 | 说明 | 包含表 |
|----------|------|--------|
| `sys_*` | 系统管理模块（RBAC） | sys_user, sys_role, sys_dept, sys_menu, sys_user_role, sys_role_menu, sys_dict_type, sys_dict_data, sys_config, sys_oper_log, sys_logininfor |
| `cf_*` | 云胶片业务模块 | cf_patient, cf_examination, cf_image, cf_diagnosis_report, cf_image_share, cf_ai_interpret |

### 1.3 业务流概述

```
患者 (cf_patient)
    └── 检查单 (cf_examination) [1:N]
            ├── 影像 (cf_image) [1:N]
            ├── 诊断报告 (cf_diagnosis_report) [1:1]
            │       └── AI解读 (cf_ai_interpret) [1:N]
            └── 影像分享 (cf_image_share) [1:N]
```

## 2. 逻辑结构设计

### 2.1 系统管理模块 (sys_*)

#### 2.1.1 用户权限管理 (RBAC)

```
sys_user ← sys_user_role → sys_role
    ↓
sys_dept (自关联树形结构)
    ↓
sys_dept (院区/科室)

sys_role ← sys_role_menu → sys_menu (菜单权限)
```

**表清单：**

| 表名 | 说明 | 关联关系 |
|------|------|----------|
| sys_user | 用户信息表 | dept_id → sys_dept |
| sys_role | 角色表 | - |
| sys_user_role | 用户-角色关联表 | user_id → sys_user, role_id → sys_role |
| sys_role_menu | 角色-菜单关联表 | role_id → sys_role, menu_id → sys_menu |
| sys_dept | 院区/部门表 | parent_id 自关联 |
| sys_menu | 菜单权限表 | parent_id 自关联 |

#### 2.1.2 系统配置

| 表名 | 说明 |
|------|------|
| sys_dict_type | 字典类型表 |
| sys_dict_data | 字典数据表 |
| sys_config | 系统参数配置表 |

#### 2.1.3 审计日志

| 表名 | 说明 |
|------|------|
| sys_oper_log | 操作日志表 |
| sys_logininfor | 登录日志表 |

### 2.2 云胶片业务模块 (cf_*)

#### 2.2.1 核心业务实体

| 表名 | 说明 | 主键类型 | 软删除 |
|------|------|----------|--------|
| cf_patient | 患者信息表 | 雪花ID (VARCHAR) | 是 |
| cf_examination | 检查单表 | 雪花ID (VARCHAR) | 是 |
| cf_image | 影像表 | 雪花ID (VARCHAR) | 否 |
| cf_diagnosis_report | 诊断报告表 | 雪花ID (VARCHAR) | 是 |
| cf_image_share | 影像分享表 | 雪花ID (VARCHAR) | 否 |
| cf_ai_interpret | AI解读表 | 雪花ID (VARCHAR) | 否 |

#### 2.2.2 检查状态流转

```
pending (待检查)
    ↓
in_progress (检查中)
    ↓
completed (已完成)
    ↓
reported (已出报告)
```

#### 2.2.3 报告状态流转

```
draft (草稿)
    ↓
pending (待审核)
    ↓
approved (已通过) / rejected (已驳回)
```

### 5.2 软删除策略

以下表支持软删除，通过 `del_flag` 字段控制：

- `sys_user.del_flag = '0'` 正常，`'1'` 已删除
- `sys_dept.del_flag = '0'` 正常，`'1'` 已删除
- `cf_patient.del_flag = '0'` 正常，`'1'` 已删除
- `cf_examination.del_flag = '0'` 正常，`'1'` 已删除
- `cf_diagnosis_report.del_flag = '0'` 正常，`'1'` 已删除
